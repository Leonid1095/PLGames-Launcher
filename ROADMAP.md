# PLGames Launcher — Roadmap / TODO

Журнал состояния и незакрытых задач. Обновлять при изменениях.

## ✅ Сделано

### Безопасность (client-side hardening)
- **XSS → RCE:** `safeUrl()` для всех серверных URL картинок (hero-слайдер, карточки, превью).
- **Path traversal:** `_safe_join()` в `set_realmlist` — отклоняет `..` и абсолютные пути из манифеста.
- **Пароль не хранится в плейнтексте:** `sso_poll` больше не пишет `password` в `plgames_settings.json`; старые пароли вычищаются при загрузке.
- **Версии:** `_ver_tuple` нормализует длину (`0.3` == `0.3.0`).
- **exe из манифеста:** запускается только bare-filename внутри папки игры.
- **Update integrity:** `_verify_update_file()` — проверка SHA256 (если сервер отдаёт `sha256`) + Authenticode-подписи перед заменой exe.

### Новости (мультипроект)
- `get_news` понимает 3 формата сервера: массив (живой `/api/launcher/news`), объект с `news`, роадмап `phases`.
- Относительные картинки (`/bg/news/x.jpg`) резолвятся к абсолютным; ISO-даты нормализуются.
- Кликабельные новости → превью-модалка (полный текст + кнопка «Открыть на сайте» через `open_external`).
- Хелперы `_project_base` / `_resolve_media` / `_shape_news_items` — каждый проект на своём хосте.

### Апдейтер
- `download_update`: сначала серверный `download_url`, при провале (404/HTML/мелкий файл/сеть) — фолбэк на последний `.exe` с GitHub. GitHub дёргается только при провале основного.

### Чистка
- Удалён мёртвый код: `launcher.py`, `config.py`, `core/`, `pages/` (в `config.py` был стухший домен `lkds-room.online`).
- Версия синхронизирована до `0.3.0` (README, installer.nsi); spec обновлён на `plgames-wow.ru`.

## 🔲 Осталось (важно)

1. **Бумпнуть версию и выпустить релиз.** На GitHub уже есть `v0.3.0` со СТАРЫМ кодом; апдейтер сравнивает по номеру, поэтому сборку «0.3.0» новее не посчитает. Чтобы доставить фиксы — поднять до **0.3.1** (app.py `LAUNCHER_VERSION`, README, installer.nsi), собрать, выпустить тег `v0.3.1`.

2. **Обновить серверный `/api/launcher/version`.** Сейчас завис на `0.2.1`, а `download_url` указывает на `..._Setup.exe`, которого нет в новых релизах (CI собирает `PLGamesLauncher.exe`). Поднять версию + поправить `download_url` на реальный ассет + желательно добавить `sha256` (код уже умеет проверять).

3. **Code signing в CI** (`build-release.yml`). Полностью закрывает RCE через неподписанное авто-обновление. Пока подпись не настроена — `_verify_update_file` ловит только повреждение/подмену подписанного бинаря, но не unsigned-подмену.

4. **Обновление в Program Files.** In-place `.bat`-swap (`move /Y`) падает без прав. Варианты: ставить в `%LOCALAPPDATA%` или запускать update-bat с UAC-elevation. Портативная установка обновляется нормально.

5. **Проверить ассет v0.3.0** (14.5 МБ против 69 МБ у v0.2.2) — возможно неполная сборка без бандла webview-данных. Убедиться, что запускается, прежде чем направлять апдейтер.

6. **#5 креды в query-string.** `fetch_credentials` шлёт `username`/`account_id` GET-параметрами — перевести на POST (нужна поддержка сервера).

7. **aria2c.exe supply chain.** Закоммичен бинарь — зафиксировать/задокументировать SHA256 источника.

8. **(Опц.) Полная независимость от GitHub.** Хостить update `.exe` на своём сервере, GitHub оставить фолбэком.

## 📌 Факты о сервере (plgames-wow.ru)
- `GET /api/launcher/news` → массив `[{tag,title,text,image(относ.),link,date(ISO)}]` — живой фид новостей.
- `GET /api/status` → `{online,accounts,online_alliance,online_horde,...}`.
- `GET /api/content/phases` → роадмап контента (объект с `phases`), НЕ новости.
- `GET /api/launcher/version` → `{version,download_url,changelog,html_url}` (на момент записи `0.2.1`, устарел).
- `GET /api/launcher/manifest` → **404** (манифеста нет; лаунчер работает на локальном `PROJECTS` в app.py).
